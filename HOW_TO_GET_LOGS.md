# Как получить логи для отладки

## На Render

1. Войдите в ваш аккаунт на [Render.com](https://render.com)
2. Откройте ваш сервис (Web Service)
3. Перейдите во вкладку **"Logs"**
4. Вы увидите все логи в реальном времени
5. Для поиска ошибок используйте фильтр или прокрутите вниз

## Локально (если запускаете на своем компьютере)

### Вариант 1: Логи в терминале
Просто запустите сервер и логи будут выводиться в терминал:
```bash
python server.py
```

### Вариант 2: Сохранение логов в файл
```bash
python server.py > server.log 2>&1
```
Логи будут сохранены в файл `server.log`

### Вариант 3: Просмотр логов в реальном времени
```bash
python server.py | tee server.log
```
Логи будут и в терминале, и в файле

## Что искать в логах

При ошибке в форме ищите строки с:
- `ERROR` - критические ошибки
- `Ошибка сохранения в БД` - проблемы с базой данных
- `Недостаточно данных` - не хватает данных в запросе
- `Критическая ошибка в register_guest` - общая ошибка регистрации

## Примеры ошибок

```
ERROR:api:Недостаточно данных: user_id=None, first_name=Иван, last_name=Иванов
ERROR:api:Ошибка сохранения в БД: UNIQUE constraint failed: guests.user_id
ERROR:api:Критическая ошибка в register_guest: ...
```

## Отправка логов

Если нужно отправить логи для анализа:
1. Скопируйте последние 50-100 строк из логов
2. Особенно важны строки с `ERROR` и `WARNING`
3. Включите контекст вокруг ошибки (несколько строк до и после)
