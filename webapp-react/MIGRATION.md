## Следующие шаги

### 1. Тестирование React версии

```bash
cd webapp-react
npm run dev
```

Проверить:
- [ ] Все вкладки работают
- [ ] Форма RSVP отправляется
- [ ] Яндекс карта загружается
- [ ] План-сетка загружается из API
- [ ] Анимации плавные
- [ ] Telegram WebApp интеграция работает

### 2. Сборка для продакшена

```bash
cd webapp-react
npm run build
```

Это создаст файлы в `../webapp/` которые будут раздаваться Python сервером.

### 3. Резервное копирование старой версии

Перед заменой старой версии:

```bash
cd ..
mv webapp webapp-old-backup
```

### 4. Интеграция с Python сервером

Python сервер (`server.py`) уже настроен на раздачу статических файлов из `webapp/`. После сборки React версии все должно работать автоматически.

### 5. Миграция Backend (следующий этап)

После успешного тестирования frontend, можно начать миграцию backend на Node.js:

- Express.js или Fastify для API
- TypeScript для типизации
- Сохранение интеграции с Google Sheets
- Миграция логики бота (aiogram → node-telegram-bot-api или grammy)

## Структура файлов

```
webapp-react/          # React версия (разработка)
webapp/                # Собранная версия (продакшен)
webapp-old-backup/     # Резервная копия старой версии
```

## Откат

Если что-то пошло не так:

```bash
rm -rf webapp
mv webapp-old-backup webapp
```

И перезапустить Python сервер.
